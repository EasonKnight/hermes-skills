# 涨跌停 0.5% 容差修复 (2026-05-15)

## 根因

`close` 从 CSV 读取时使用 `dtype="float32"`，而 `limit_up` 由 `np.round(prev * 1.10, 2)` 计算得到 **float64**。精确 `==` 比较时，float32 精度误差导致绝大多数涨停被漏算。

### 精度差异示例

```python
np.float32(14.26) == np.float64(14.26)  # → False!
print(np.float32(14.26))  # 14.260000228881836
print(np.float64(14.26))  # 14.26
```

### 影响范围

| 指标 | 精确匹配 `==` | 0.5%容差 `>= limit_up * 0.995` |
|------|:------------:|:-----------------------------:|
| 涨停拦截 | 2,529次/10年 | 109,553次/10年 |
| 跌停拦截 | 969次/10年 | 55,235次/10年 |
| 前500天漏算 | — | 9,998次float32精度导致漏算 |

### 修复代码

```python
# _compute_limit_status() 中
# 之前（漏算99%）：
self.limit_up_hit[:, t] = (c == self.limit_up[:, t]) & (c > 0) & ~self.suspended[:, t]

# 之后（正确拦截）：
self.limit_up_hit[:, t]   = (c >= self.limit_up[:, t] * 0.995) & (c > 0) & ~self.suspended[:, t]
self.limit_down_hit[:, t] = (c <= self.limit_down[:, t] * 1.005) & (c > 0) & ~self.suspended[:, t]
```

1. **动量/涨幅类策略在A股回测中需谨慎** — 它们的名义收益来自能买入涨停板股票，但实战中涨停买不到
