# 策略平台大扫除方法论

## 背景

本次迭代从 **69 个策略文件** 精减到 **25 个**，删除 44 个，保留率 36%。方法可复用。

## 裁剪原则

### 1. 分类整理

```
分类 → 同类保留最佳 → 删除其余
```

具体分类：基线策略、低价策略、反向策略、动量+低波混合、辅助参考。

### 2. 删除条件

| 条件 | 例子 | 处理 |
|------|------|------|
| 年化 <-5% 或 夏普 <-0.3 | S22 (-61.45%), S16 (-55.39%) | 立即删除 |
| 年化 < 5% 且无创新性 | S73 (2.72%), S71 (2.31%) | 删除 |
| 与强者完全重复 | S59=S60(布林中轨=MA20), S26=S68(高价股) | 去重留一 |
| 被更强变体覆盖 | S25→S66, S87→S78 | 删旧留新 |
| 相似度过高 | S63/S64/S65 都是低价变体但S66最优 | 只留S66 |

### 3. 保留条件

- 年化 > 15% 且夏普 > 0.4 → 核心保留
- 年化 10-15% 且策略类型独特（如均值回归、双底反弹）→ 辅助参考保留
- 夏普 > 0.8 即使年化一般（如S57超涨反向夏普1.06）

## 执行流程

```bash
# 1. 先分析——用 Python 扫描所有策略文件 + 读取回测 stats
# 2. 分类——按策略类型分组，组内排序
# 3. 标记删除——确定每个组保留谁、删除谁
# 4. 批量删除——rm strategies/s*.py
# 5. 清理结果目录——rm -rf results/SXX-*/
# 6. 验证——ls strategies/s*.py | wc -l
# 7. 提交——git add -A && git commit
```

## 注意

- 删除策略文件后必须同步删除对应 `results/<folder>/`
- `.gitignore` 已排除 `results/`，但删除策略文件后 `git status` 仍能正确跟踪
- git 可能检测到 renames（不同文件内容相似度匹配），不影响实际内容
